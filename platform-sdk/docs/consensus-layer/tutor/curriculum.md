## Authoring notes (reviewer decisions baked in)

- **`quiescence` → included as `a4-05-quiescence` (Event Creator).** Quiescence is an opt-in veto that pauses self-event creation when nothing is pending and no deadline is near, so it sits with the Event Creator's other gates, before the cluster synthesis. **The `quiescence.md` topic file is currently a 57-line all-`[TBD]` stub and must be fleshed out before this lesson is authored** — otherwise the lesson would itself be all `[TBD]`. It sits at index 32, so flesh out the topic before the authoring loop reaches it (mid–Cluster A.4). It also constrains how stale events are reported across the Consensus/Execution boundary, which links it to `a1-07-stale-events`.
- **`iss-detection` folded into Cluster C** as `c-03-iss-detection`. It is a substantive topic file and a sibling of signed-state (signed-state-management's "ISS detection" section routes every hashed state to it; restart-and-pces covers offline ISS recovery via ADR-003), and Pass 3 edge 04 depends on it. It is not among the brief's eleven named topics, so this insertion is a reviewer call — fold into `c-01`/`c-05` if a leaner Cluster C is preferred.
- **Cluster A.1 is the densest cluster (8 entries).** Splitting `a1-03-strongly-seeing` from `branching`, or `a1-05` into separate judges and consensus-order lessons, are the natural expansion points; merging `a1-04` (voting) into adjacent lessons is the natural trim.
- **Cluster 0 has no `concepts/` anchors.** Its breakdown follows the wiring topic's section structure (schedulers/types, wires+soldering, backpressure+health wire, WiringModel lifecycle) and the brief's coverage list; it leans on the glossary entries for wire, scheduler, soldering, transformer, squelching.
- **KB-gap notes for Phase 2** (do not affect this manifest): `../delta-map/` holds only a README — every `delta-map/<topic>.md` is absent, so lesson Delta-callout sections will emit `[TBD]` until they land. `../invariants/` is a per-file directory (only INV-001 exists), not the single `invariants.md` the prompt names. Confirmed catalog anchors lessons can cite today: RUL-001 (signed-state reservations → `c-02`), ADR-002 (freeze signature handoff → `d-03`), ADR-003 (offline ISS recovery → `c-05`), ADR-001 (PCES/state-snapshot coordination → `c-04`), SCN-001 and INV-001 (round-created monotonicity → A.1).
