# Delta map: consensus-boundary

## Summary

The proposal recasts Consensus as a library that Execution constructs and tears
down through a single public API: a top-level API/impl module pair, one unified
API surface, and the `initialize` / `destroy` instance lifecycle. None of this
overarching structure has begun — there is no top-level Consensus module, the
surface is split across `ExecutionLayer` and `ConsensusStateEventHandler`
(registered through `PlatformBuilder`), and the platform, not Execution, owns the
instance lifecycle. The per-method seam features — the `nextRound` pull and the
callbacks — live in their topic files, where each shapes that topic's
architecture.
